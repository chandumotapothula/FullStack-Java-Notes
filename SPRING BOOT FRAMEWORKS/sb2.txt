Spring boot Starters 
====================
It is one of the component of spring boot.

It is used to simplify dependency management by grouping common dependencies in a single starter.

Spring boot contains many built-in starters to develop applications rapidly and easily.

It is also known as dependency descriptor.

All spring boot built-in starters follow below naming pattern.

syntax:
	spring-boot-starter-* 

	Here '*' means name of the application
ex:
	spring-boot-starter-web 
	spring-boot-starter-thymeleaf 
	spring-boot-starter-data-jpa
	spring-boot-starter-validation 
	spring-boot-starter-security 
	and etc.



Spring Web Dependency 
=====================
Spring Web dependency is used to develop web applications in spring boot.

It contains REST,Spring MVC and Tomcat as a embedded server.

To develop web application we need to add below dependency inside pom.xml file.
ex:
	<dependency>
    		<groupId>org.springframework.boot</groupId>
    		<artifactId>spring-boot-starter-web</artifactId>
    	</dependency>

There are two important features of spring web dependency.

1) It is compatible for web application.

2) It performs autoconfigurations.

Spring web dependency performs following things as autoconfigurations.

1) DispatcherServlet 

2) ErrorPage

3) Manage static dependent dependencies

4) Servlet Container

step1:
------
	Create a spring boot starter project i.e SecondSB.
	ex:
		starters:
				Spring Web 

step2:
-----
	Add tomcat jasper dependency inside pom.xml file.
	ex:
		<dependency>
    			<groupId>org.apache.tomcat.embed</groupId>
    			<artifactId>tomcat-embed-jasper</artifactId>
   		</dependency>

step3:
------
	Create a "webapp" custom folder inside "src/main" folder.

step4:
------
	Create a "index.jsp" file inside "webapp" folder.

index.jsp 
---------
<center>
	<h1>
		I Love Spring Boot 
	</h1>	
</center>

step5:
------	
	Create a "com.ihub.www.controller" package inside "src/main/java" folder.

step6:
------
	Create a HomeController.java file inside "com.ihub.www.controller" package.
	
HomeController.java
--------------------
package com.ihub.www.controller;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
@Controller
public class HomeController 
{
	//handler method 
	@RequestMapping("/msg")
	public String home()
	{
		return "index.jsp";
	}
}

step7:
------
	Configure application name and server port in application.properties file.

application.properties 
-----------------------
spring.application.name=SecondSB
server.port=9090


step8:
------
	Run spring boot starter project.

step9:
------
	Test the application by using below request url.
	ex:
		http://localhost:9090/msg

Interview Questions 
====================

Q) What is @Controller annotation?

It is a class level annotation.

It is a stereotype annotation.

It indicates that a perticular class serves as a controller and takes all incoming requests. 


Q) What is @RequestMapping Annotation?

It is a class level and method level annotation.
It is used to handle all HTTP methods i.e GET, POST, PUT, DELETE and etc.
It is used to map the request to controller or handler method.


Q) What is @ResponseBody annotation?

It is a method level annotation.
It tells to the controller that return object automatically serialized to Text, JSON or XML.
